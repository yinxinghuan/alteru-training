Part 8 · Game Template Line

一句话/一张照片
出一个有质感的游戏

痛点：平台普通用户做不出能玩的游戏。不是缺想法，是缺质感——juice、首屏即玩、UI、排行榜、社交通知……这些层门槛太高。

解法：锁死质感 = 做不烂。

一句话/一张照片 → Cartridge（卡带 JSON）→ 注入锁死 Engine → 可玩游戏

Engine（引擎）——我们造，用户碰不到。
永远自带：juice / instant-play / 排行榜 / 留言 / 通知。

Cartridge（卡带）——用户的一句话生成。
只含主题：敌人叫什么、长什么样、什么颜色、什么文案。
任何会改变"游戏多难"的字段都不准进卡带 → 做不烂。

Engine #1：Block Party
→ 猫 vs 扫地机器人（换一份卡带 JSON）

Part 8 · Game Template Line

One sentence. One photo.
A quality game comes out.

The pain point: Regular users on the platform can't make playable games. Not a lack of ideas — a lack of quality. Juice, instant-play, UI, leaderboards, social notifications... the barrier is too high.

The fix: lock quality so they can't break it.

One sentence → Cartridge (JSON) → locked Engine → playable game

Engine — We build it. Users can't touch it.
Always includes: juice / instant-play / leaderboard / guestbook / notify.

Cartridge — Generated from the user's one sentence.
Theme only: enemy names, visuals, colors, copy.
Nothing that changes difficulty can enter the cartridge → they can't break it.

Engine #1: Block Party
→ Cat vs Robot Vacuums (swap one cartridge JSON)

讲师备忘 · 按 S 关闭Speaker Notes · S to close

先给学员看痛点数据。平台 100+ 游戏但多数是 Tier1 纯 HTML——根本没质感。然后引出核心洞察：质感不在玩法复杂度，而在 juice/UI/社交那几层，而这几层我们已经一条条沉淀成了 skill。

Engine/Cartridge 类比红白机：机器永远不变，插不同卡带玩不同游戏。关键设计：卡带不含任何游戏难度字段——HP/速度/碰撞全锁在引擎里。

Start with the pain point. 100+ games on the platform, most Tier1 bare HTML — no quality. The insight: quality isn't gameplay complexity, it's juice/UI/social layers. We've already codified those into skills. Engine/Cartridge = NES analogy: machine never changes, different cartridge = different game. Critical design: cartridge can't contain difficulty fields.
