Part 8 · 如何使用

一条命令，
一个游戏

Generator 封装了整个流程：

  1. LLM 拼 prompt——7 段 system prompt 教 LLM 产出合法 CartridgeSpec（schema/枚举/palette 指南/完整范例）
  2. JSON 提取+校验——4 层回退策略 + validateSpec()，不合法自动重试带错误纠正
  3. （可选）gen-image sprite——6 种敌人各调 txt2img 出图→R2 上传→引擎 billboard 渲染
  4. swap + build——改 cartrdge/index.ts 一行 → 新游戏

当前可选：--sprites（gen-image 出敌人 sprite）、--dry-run（预览）、--retries N

$ npx tsx scripts/gen-cartridge.ts \
  --sentence "a cat surviving robot vacuums" \
  --sprites

→ LLM outputs CartridgeSpec JSON
→ txt2img × 6 for enemy sprites
→ Upload to R2 for permanent storage
→ Write cartridge/gen-cat-vacuum.ts
→ Swap index.ts → npm run build
→ 🎮 FUR FLOOD is playable

One sentence → playable game, fully automatic ~60 seconds

Part 8 · How to Use

One command,
one game

The generator packages the entire pipeline:

  1. LLM prompt assembly — 7-section system prompt teaches the LLM to output valid CartridgeSpec (schema/enums/palette guide/worked example)
  2. JSON extraction + validation — 4 fallback strategies + validateSpec(), auto-retry with error feedback
  3. (Optional) gen-image sprites — txt2img for 6 enemy types → R2 upload → engine billboard rendering
  4. Swap + build — change one line in cartridge/index.ts → new game

Flags: --sprites (gen-image enemy art), --dry-run (preview), --retries N
